🔐 2. Flujo de Autenticación

A. Paso Previo: ¿Login o Registro?

Para evitar preguntarle al usuario si quiere registrarse o iniciar sesión, puedes usar este endpoint:

POST /api/auth/check-email 
// Body Request
{ "email": "usuario@ejemplo.com" }

// Response
{
  "success": true,
  "data": {
    "exists": true // true -> Login | false -> Registro
  }
}

B. Login y Registro (Optimizados)

Para mantener la carga útil ligera y segura, estos endpoints ahora solo devuelven tokens, y NO incluyen la información del usuario.

POST /api/auth/login y POST /api/auth/register 
// Response
{
  "success": true,
  "data": {
    "accessToken": "eyJhbG...",
    "refreshToken": "eyJhbG..."
  }
}
💡
Almacenamiento Seguro Debes guardar el accessToken y refreshToken en el almacenamiento seguro del dispositivo (ej. SecureStore en Expo).

C. Recuperando el Perfil y los Hogares

Una vez iniciada la sesión, debes llamar a este nuevo endpoint para obtener toda la información que necesita la UI inicial:

GET /api/users/me

Headers: Authorization: Bearer {accessToken}

// Response
{
  "success": true,
  "data": {
    "user": {
      "id": "6a381f...",
      "email": "usuario@ejemplo.com",
      "name": "Elias Duarte",
      "plan": "free"
    },
    "households": [
      {
        "id": "abc123",
        "name": "Mi Hogar",
        "role": "admin",
        "is_owner": true,
        "invite_code": "XYZ789", 
        "members_count": 3
      }
    ]
  }
}
⚠️
Lógica de Redirección basada en Households
  • Si households está vacío ([]) → Llevar al Onboarding de Hogares.
  • Si is_owner es true → Mostrar el botón "Compartir Código" con invite_code.
  • Si is_owner es false → invite_code vendrá como null.

🚦 3. Resumen del Flujo en la App

  • App Arranca

    ¿Hay token guardado?

    • SÍ: Llama a GET /api/users/me. Si 200 OK -> Home. Si 401 -> Intenta Refresh -> Si falla -> Ingresar Email.
    • NO: Ve a pantalla Ingresar Email.
  • Pantalla Ingresar Email

    Llama a POST /api/auth/check-email

    • Si exists: true -> Solicitar Contraseña (Login)
    • Si exists: false -> Completar Datos (Registro)
  • Al loguearse / registrarse
    • Guarda tokens en SecureStore.
    • Llama a GET /api/users/me.
    • Evalúa hogares: ¿Vacío? -> Onboarding. ¿Lleno? -> Home.
